# SentinelX

AI-Native Cyber Intelligence & Autonomous Defense Platform

![SentinelX Banner](./public/banner.png)

## Overview

SentinelX is an AI-driven cyber intelligence and autonomous defense platform designed to simulate, visualize, analyze, and eventually orchestrate intelligent responses to cybersecurity threats in real time.

The platform combines real-time telemetry visualization, graph-based infrastructure intelligence, replay systems, AI-assisted threat analysis, and operational SOC-style workflows into a unified cybersecurity operations environment.

Unlike traditional log-heavy SIEM dashboards, SentinelX focuses on contextual cyber situational awareness through interactive infrastructure visualization, telemetry-driven event processing, and graph-native threat analysis.

The project is currently being developed as a scalable MVP architecture intended to evolve toward enterprise telemetry ingestion, AI-assisted operational reasoning, and autonomous defense systems.

# System Architecture

```txt
Real Infrastructure / Simulated Telemetry
                ↓
       Telemetry Ingestion Layer
                ↓
        Event Processing Engine
                ↓
        WebSocket Streaming Layer
                ↓
         SentinelX Frontend
                ↓
 Graph Intelligence + AI Reasoning
```

SentinelX follows an event-driven architecture where all operational systems communicate through centralized telemetry events.

This separation allows:

* replay reconstruction,
* real-time synchronization,
* scalable telemetry ingestion,
* modular AI integration,
* and future enterprise extensibility.

# Current MVP Scope

The current MVP focuses on validating:

* telemetry-native frontend architecture,
* graph-based infrastructure visualization,
* real-time attack propagation,
* replay system concepts,
* AI-assisted operational analysis,
* and scalable cybersecurity workflow design.

The platform currently operates using simulated telemetry events generated by the internal simulation engine.

The long-term architecture is designed so that simulated telemetry can later be replaced by real enterprise telemetry sources without major frontend architectural changes.

# Why SentinelX?

Traditional cybersecurity systems often rely heavily on disconnected logs, fragmented alerts, and limited infrastructure context.

SentinelX approaches cybersecurity from a graph-native and telemetry-centric perspective where infrastructure relationships, attack propagation, contextual analysis, and operational workflows are unified into a single interactive system.

The project is heavily inspired by:

* modern SOC environments,
* cyber situational awareness systems,
* graph intelligence platforms,
* observability architectures,
* and AI-assisted operational tooling.
